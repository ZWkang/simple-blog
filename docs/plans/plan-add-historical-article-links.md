# 在 About 页面添加历史文章平台链接

## Context
用户在掘金(Juejin)和思否(SegmentFault)上有一些历史文章,希望在 about 页面添加描述和链接,方便访客查看他们的早期作品。

## 目标
在 `app/about/page.mdx` 中添加:
1. 一段描述文字,说明在其他平台有历史文章
2. 掘金和思否的链接

## 实施计划

### 1. 内容位置
在现有介绍段落之后、社交链接之前添加新段落

### 2. 文案风格
- 使用中文,与现有内容保持一致
- 简洁自然,不过于正式
- 说明这些是早期/历史文章

### 3. 链接样式
两种方案:
- **方案 A**: 内联文本链接(推荐) - 更简洁,适合"历史文章"这种次要信息
- **方案 B**: 添加到社交图标区域 - 需要找到掘金和思否的 SVG 图标

推荐方案 A,因为:
- 实现更简单
- 这些是历史内容,不需要与当前社交账号同等突出
- 保持页面简洁

### 4. 具体修改
在第 7 行(博客介绍段落)之后添加:

```markdown
早期我也在 [掘金](https://juejin.cn/user/4037062425593447/posts) 和 [思否](https://segmentfault.com/u/zwkang) 上写过一些文章,欢迎查看。
```

或者更详细的版本:

```markdown
在这个博客之前,我也在 [掘金](https://juejin.cn/user/4037062425593447/posts) 和 [思否(SegmentFault)](https://segmentfault.com/u/zwkang) 上分享过技术文章,感兴趣可以看看早期的内容。
```

### 5. 关键文件
- `app/about/page.mdx` - 唯一需要修改的文件

## 验证
1. 运行 `npm run dev` 查看页面效果
2. 确认链接可点击且正确跳转
3. 检查文案是否自然流畅
4. 确保 `target="_blank"` 和 `rel="noopener noreferrer"` 属性(MDX 会自动处理外部链接)

## 实施结果

✅ **已完成** (2026-02-25)

在 `app/about/page.mdx` 第 9 行添加了历史文章平台链接段落:

在这个博客之前，我也在 [掘金](https://juejin.cn/user/4037062425593447/posts) 和 [思否(SegmentFault)](https://segmentfault.com/u/zwkang) 上分享过技术文章，感兴趣可以看看早期的内容。

- 采用方案 A (内联文本链接)
